## [0.4.16] - 2026-06-13

### 🔧 优化
- **主精简字数翻倍放开**：`_capsule_size_tier` 档位从 40/80/150/200 → **80/150/280/400**，条数 2/3/5/7 → 3/4/6/8，每条字数 20/25/30/35 → 30/40/50/55。原档位短输入时常把关键信息切丢（用户实测「3 句话精简后只剩 1 句」），新档位接近 flomo 卡片的舒适阅读量
- **视角补充字数放开**：硬上限 150 → **250 字**，最多 2 → **3 条**。够放下完整一组「📌 新事实 + 📚 KB 事实 + 🔗 因果链」拼图，不再因 150 字砍掉「📚」那截。`CAPSULE_ASIDE_HARD` 兜底 180 → 290（含 prompt 内字面量同步更新）
- **「📚 笔记」开关切换后不再自动重精简**：原行为一切就 `runRefine()` 烧 token + 等 8-15s，对来回比较两个模式的用户体验极差。改成：toggle 完只更新状态（按钮变 ON/OFF + 内部状态持久化），同时在 result 卡下方浮出一个虚线 chip「↻ 用「笔记 ON」重算（会更慢，含视角补充）」；用户主动点 chip 才真的重跑。开新一轮（runRefine）时 chip 自动隐去

## [0.4.15] - 2026-06-13

### ✨ 新增
- **图片即附件归档（flomo 风格）**：双击 ⌘C 复制图片 → 胶囊弹出图片预览（**不调 AI**），点「📥 归档+复制」才真的归档。图片落到 KB 文件**同级**的 `attachments/` 目录（文件名 `2026-06-13-155234.png`），KB 文件追加 markdown 引用 `![](attachments/2026-06-13-155234.png)`（相对路径，跟 KB 文件一起走）。归档后剪贴板里**仍然是原图**，照样能粘到别处。文件名秒级冲突自动加 `_2 _3` 后缀；attachments/ 不存在自动创建
- **图片读剪贴板**：菜单栏新增 `_read_pb_image()`，优先 `NSPasteboardTypePNG`，没有时 `NSPasteboardTypeTIFF` → `NSBitmapImageRep` 转 PNG；不调 osascript / pbpaste。双击同图判定用「头 4KB + 尾 4KB sha256」做轻量指纹（避免大图 hash 慢）
- **图片缩略图（前端预览）**：`NSImage` + `NSBitmapImageRep` 缩到 ≤ 600 px 宽再 base64 嵌 HTML，PyObjC 自带，**零额外依赖**（拒了 PIL/Pillow）

### 🔧 优化
- **胶囊侧 `_read_clip` 改用 NSPasteboard**：v0.4.11 只改了菜单栏轮询路径，胶囊冷启动时 `quick_archive.py:_read_clip` 仍在调 `pbpaste`，剪贴板里有图片时启动会卡几秒。换成 `NSPasteboard.stringForType_(NSPasteboardTypeString)`：图片直接返回 ""，零等待
- **清理残留 dead code**：`history.py:read_clipboard_image / record_image` 是被菜单栏 import 但从未调用的死代码（osascript + 临时文件路径），已删除；图片归档完全走 v0.4.15 的新路径

## [0.4.14] - 2026-06-11

### ✨ 新增
- **BM25 KB 检索 top-3 段**：替换原「取尾部 2400 字」。新增零依赖 `kb_index.py`（中文 unigram + bigram 分词、Okapi BM25），把 KB 按 `^## ` 二级标题切段（141KB 的 KB 切 60 段、< 1ms），用本次输入做 query 召回 top-3 命中段（< 15ms）。命中精度大幅高于尾部切片：用户复制讨论 A 不会再误读到尾部最近写入的讨论 B。检索结果按 `(path, mtime)` 缓存，避免每次复制都重切大 KB
- **prefix cache 命中率 telemetry**：DeepSeek 流式响应里的 `prompt_cache_hit_tokens / *_miss_tokens` 通过新增的 `stream_options.include_usage=true` 拿到，写入 `timings.cache_hit_tokens / cache_total_tokens / cache_hit_ratio / completion_tokens`，前端 hover 时间标签可看

### 🔧 优化
- **prompt 拼接顺序重排（DeepSeek prefix cache 友好化）**：原顺序把动态内容（KB / 重复警告 / 反馈教训）夹在静态内容（profile / aside）中间，DeepSeek 命中前缀只能截到 `head`（~1300 字）。重排后顺序变为 `head + boundary_decl + profile + aside + lessons + dup + kb_block`，把所有静态 / 半静态前置、所有动态内容（每次都变的 KB BM25 节选）放最末尾，命中段长度从 ~1300 字 → ~5000 字（带档案场景）。命中后 input token 单价 0.26x、TTFB 显著下降。配套修订 `boundary_decl` 措辞：把末尾的「笔记上下文」也声明为「仅用于视角补充段」（KB 用途从「精简正文术语对齐 + 视角补充事实来源」收紧到「仅视角补充事实来源」，换前缀稳定性）

## [0.4.13] - 2026-06-10

### 🐛 修复
- **双击 ⌘C 几乎永远不响应**：原 500ms 窗口与 500ms poll 周期同量级，单 poll 内捕获两次 changeCount 跳变的概率极低；窗口默认改 **1500ms**，覆盖用户实际"连按"节奏（0.8-1.5s）
- **双击文本要求过严**：原要求两次复制内容完全相同，实际选区常有 1-2 字微调；改为「完全相同 OR 长度差 ≤5% 且前 60 字 prefix 相同」
- **打包后 .app 藏在临时目录里**：`release.sh` 用 `ditto --keepParent` 把 `mktemp -d` 的目录名一并打进 zip 顶层，解压后 `~/Applications/Skillless.app` 不存在；去掉 `--keepParent`，zip 顶层直接是 `Skillless.app` / `README.md` / `CHANGELOG.md`
- **single 日志加诊断字段**：`copy single` 行新增 `delta=N elapsed=Nms match=bool`，方便排查"连按几次还不弹"

## [0.4.12] - 2026-06-10

### 🔧 优化
- **精简正文字数软上限**：按输入长度分档（≤100 字→总 ≤40 字 / ≤300 字→≤80 字 / ≤1500 字→≤150 字 / 更长→≤200 字），允许超出至多 20%，prompt 里明确「严禁展开成步骤列表 / 部署清单 / 小作文」。复活了一直没接线的 `_capsule_size_tier`，极速 / 完整模式都按档位生成条数 + 每条字数 + 总字数约束

### 🐛 修复
- **笔记 OFF 仍渲染「💡 视角补充」卡**：双保险修复。① 后端极速模式 prompt 此前仍带「`---` + 第二段：视角补充」两段式指令，导致模型自发输出视角补充段——现在极速模式（含无 KB 上下文时）改用单段式指令并硬禁止输出 `---` 与「视角补充」段；② 前端 `setResult` 在 `kb_enabled=false` 时无条件丢弃 `---` 之后的内容（内部记录到 `state.dropped_aside`），流式中间帧与最终渲染同一路径 gating
- **胶囊下方大片空白**：autofit 此前用 `body.scrollHeight` 量内容高，但 `.body` 是 flex 拉伸子项——窗口被流式中间帧撑高后 scrollHeight 等于被撑大的高度，「只涨不缩」。改为测量子元素包络高度（首/末子元素 boundingRect + padding），流式收尾后窗口能正确回缩；同时把反馈条 / 疑似拼写卡的高度计入，避免被裁切

## [0.4.2] - 2026-06-08

### ✨ 新增
- **重复输入吐槽**：如果你复制的内容和最近 30 天精简过的某次几乎一样（sha1 全等或头尾哈希+长度匹配），AI 视角补充开头会加一句同事级吐槽（"这段你 X 分钟前刚精简过——还是同一个建议"），然后继续正常精简
- 胶囊标题展示「N 分钟前重复」角标，方便你知道为啥被吐了

### 🔧 优化
- **字数死命令**：精简正文硬上限 350 字（不管输入多长）；视角补充 ≤ 150 字、最多 2 条；超出由后端 `_enforce_length_limit` 兜底截断（智能边界 + 「…」），标题加 `· 已收短` 角标提示。`max_tokens` 700→600

### 🐛 修复

### 🔧 优化
- **作用域隔离**：SOUL/USER/TOOLS 三份档案**只服务视角补充段**，精简正文按基础规则走、不带姿态、不模仿用户文风、不引用人设描述（prompt 里三次重复边界声明，模板顶部也明确）

- **纯精简模式**（无 KB 默认文档）完全不注入档案，零行为变化
- **胶囊里的文字（精简正文、视角补充、疑似拼写卡片、反馈条）现在可以部分选中复制**：选中后系统级 ⌘C 走标准复制；未选中时仍是「复制全部精简结果」原行为
