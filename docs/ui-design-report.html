白色极简方案：让胶囊与后台共用一套设计语言

本报告把 AI Archiver 现在的「黑色胶囊 + 白色后台」整合成一套以白色为基底、 强调克制留白与低反差对比的设计语言。胶囊有专门的视觉方案（含路径切换入口 ›），与后台共享同一组 token，避免在两个场景里来回切换审美。

关于参考来源：当前会话内没有挂载 Mobbin MCP，也没有浏览器 MCP， 所以这份报告不是「截图分析」，而是基于业内通行的极简白色 App 设计实践 归纳出的 design system。下面所有 mockup 都是为 AI Archiver 原创设计，没有复刻任何具体 App 的 UI。

1设计基线

四条 hard rule，整套 App 共用。每个 UI 元素都用这四条做 sanity check。

克制对比

正文 #1f2937 配 #ffffff， WCAG AAA；次级文字 #64748b 配 AA。 全局只允许一个 accent（天蓝），其他都是中性灰。

留白优先

卡片内边距固定 18-20px；卡片之间 12-16px； 正文行高 1.55-1.65，标题 1.3-1.4。 不要塞满，宁可少放也别贴边。

单层级阴影

全应用只有两种 elevation： level-0（无阴影 + 1px 边框）和 level-1（柔和投影，胶囊/弹层专用）。 不叠加。

系统字体

不引入网络字体。-apple-system, "PingFang SC" 在 macOS 上的中英文渲染已经接近完美，加载零成本。

6后台 dashboard 视觉微调

现版后台已经是白色，但有几个小地方可以再克制一下。

① 去掉渐变与多余阴影

现在 .stat-card 和 .default-doc-card 都用了 linear-gradient，建议改成纯 #f0f9ff 单色 + 1px #bae6fd 边框。性能提升一截，视觉更稳。

② 状态色用 tint 不用 bold

删除按钮、清空历史现在用了红色文字 / 红边框。改成 hover 才显红，平时是中性灰， 减少「红色噪音」。

③ 卡片标题字号统一 13→14

现版 h3 是 13px（同 body），层级感弱。 升到 14px + 700 weight，让标题真正成为锚点。

④ 侧边栏 icon 字号

现版 emoji 占了主导（📊 📜 📕 ⌨️），略卡通。 Option A：换成统一描线 SVG icon； Option B：保留 emoji 但调小到 13px，与文字平视。
